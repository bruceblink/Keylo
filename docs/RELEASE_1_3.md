# Keylo 1.3.0 发布说明

发布日期：2026年5月

## 版本定位

Keylo 1.3.0 是在 1.2.x 安全修复基础上的功能增强版本，重点聚焦：

1. 认证与授权链路的高风险缺陷修复
2. 运维可观测性增强（日志双写到控制台与本地磁盘）
3. 发布稳定性与回归验证完善

本版本无破坏性 API 变更，支持平滑升级。

---

## 主要改进

### 🔐 安全与正确性修复

#### 禁用用户不可再签发访问令牌
`/v1/auth/token` 用户密码登录链路补充 `active` 检查，`active=false` 用户即使密码正确也会被拒绝。

- 保持与错误密码一致的失败语义，避免账号枚举风险。
- 相关回归测试已新增并通过。

#### 管理客户端权限语义收敛
创建管理客户端时统一写入 `is_admin_client=TRUE`，与 `/v1/admin/token` 授权判定逻辑一致，避免“创建成功但无管理权限”的行为偏差。

#### CORS 来源校验回归修复
严格来源校验中，允许合法 Origin 在 URI 解析后出现的默认 `/` 路径（如 `https://example.com/`），并继续拒绝带非根路径或 query 的来源。

---

### 📈 可观测性增强

#### 新增日志双输出（console + file）
服务在保留控制台日志输出的同时，支持将日志写入本地磁盘目录（容器场景下可落盘到宿主机）。

新增配置项：

| 变量 | 说明 | 默认值 |
|------|------|------|
| `LOG_TO_FILE` | 是否启用文件日志输出 | `true` |
| `LOG_DIR` | 日志目录 | `./logs`（容器推荐 `/app/logs`） |
| `LOG_FILE_PREFIX` | 日志文件前缀 | `keylo` |
| `LOGS_DIR` | Docker 挂载到宿主机的日志目录 | `./logs` |

实现细节：
- 使用 `tracing-appender` non-blocking writer，降低日志 I/O 对请求路径的影响。
- docker-compose 新增日志目录挂载：`${LOGS_DIR:-./logs}:/app/logs`。

## 升级指南

### 从 1.2.x 升级到 1.3.0

1. 更新镜像/二进制到 `v1.3.0`。
2. 若使用 Docker Compose，建议设置并确认：

```env
LOG_TO_FILE=true
LOG_DIR=/app/logs
LOG_FILE_PREFIX=keylo
LOGS_DIR=./logs
```

3. 确认宿主机日志目录可写（如 `./logs`）。
4. 重启服务并验证：
   - `docker compose logs keylo` 有控制台日志
   - 宿主机 `./logs` 出现滚动日志文件
